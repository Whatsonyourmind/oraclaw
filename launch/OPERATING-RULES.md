# OraClaw Operating Rules

These rules are permanent. Every session must follow them.

## NEVER Do

1. **NEVER include product links, curls, or package imports in GitHub comments.** Pure math only.
2. **NEVER comment on banned repos/orgs** (see list below).
3. **NEVER post on closed issues or maintainer-only threads.**
4. **NEVER open issues to promote OraClaw on other repos.**
5. **NEVER create new npm packages or algorithm features** until first revenue.
6. **NEVER publish articles on Dev.to, Hashnode, or Medium** until first revenue (zero ROI at current scale).

- Pure math expertise on 10K+ star repos -> profile clicks -> repo visits -> stars
