## Key Architectural Decisions

1. Dual billing paths (Stripe + x402) at launch for maximum revenue capture
2. Fire-and-forget meter events (.catch() logs but never blocks API response)
3. Hook factory pattern (createXHook) for testability with injected dependencies
4. Direct function dispatch for batch (not app.inject) to avoid hook re-execution
5. Promise.allSettled for batch parallelism with per-call inline RFC 9457 errors
6. Pre-compiled JS over tsx runtime to cut cold-start by 2-5s on Render free tier
7. Loose X402Server interface instead of importing @x402/core types for decoupling
