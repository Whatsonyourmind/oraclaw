  <action>
1. Create `mission-control/scripts/publish-all.sh` — a local publish script that:
   - NOTE: This script supersedes the existing `scripts/retry-npm-publish.sh` (which was a quick fix for a subset of packages with no version-check or summary). Delete `scripts/retry-npm-publish.sh` after confirming publish-all.sh works.
   - Iterates over all 14 SDK packages in `packages/sdk/*/`
   - For each package: reads name and version from package.json, checks if that version is already published via `npm view "$name@$version" version 2>/dev/null`, skips if already published, otherwise runs `cd "$pkg_dir" && npm run build && npm publish --access public` with up to 3 retries on failure
   - After SDK packages, does the same for `packages/mcp-server/`
   - Prints a summary of published/skipped/failed packages at the end
   - Make the script executable (`chmod +x`)

2. Create `mission-control/.github/workflows/publish-packages.yml` — GitHub Actions OIDC Trusted Publishing workflow:
   ```yaml
   name: Publish Packages
   on:
     workflow_dispatch:
       inputs:
         dry-run:
           description: 'Dry run (skip actual publish)'
           required: false
           default: false
           type: boolean

   permissions:
     contents: read
     id-token: write

   jobs:
     publish:
       runs-on: ubuntu-latest
       defaults:
         run:
           working-directory: mission-control
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: '22'
             registry-url: 'https://registry.npmjs.org'
         - name: Upgrade npm for trusted publishing
           run: npm install -g npm@latest
         - name: Build and publish SDK packages
           run: |
             for pkg_dir in packages/sdk/*/; do
               name=$(node -p "require('./$pkg_dir/package.json').name")
               version=$(node -p "require('./$pkg_dir/package.json').version")
               published=$(npm view "$name@$version" version 2>/dev/null || echo "")
               if [ "$published" = "$version" ]; then
                 echo "SKIP $name@$version (already published)"
                 continue
               fi
               echo "Publishing $name@$version..."
               (
                 cd "$pkg_dir"
                 npm run build
                 if [ "${{ inputs.dry-run }}" = "true" ]; then
                   echo "DRY RUN: would publish $name@$version"
                   npm pack --dry-run
                 else
                   npm publish --provenance --access public
                 fi
               )
             done
         - name: Build and publish MCP server
           run: |
             cd packages/mcp-server
             name=$(node -p "require('./package.json').name")
             version=$(node -p "require('./package.json').version")
             published=$(npm view "$name@$version" version 2>/dev/null || echo "")
             if [ "$published" = "$version" ]; then
               echo "SKIP $name@$version (already published)"
             elif [ "${{ inputs.dry-run }}" = "true" ]; then
               npm run build
               echo "DRY RUN: would publish $name@$version"
               npm pack --dry-run
             else
               npm run build
               npm publish --provenance --access public
             fi
   ```

   Key details:
   - `permissions: id-token: write` enables OIDC token exchange with npm
   - `workflow_dispatch` for manual triggering (not every push)
   - `--provenance` flag generates OIDC-based provenance attestations
   - `--access public` required for scoped packages on first publish
   - npm@latest install ensures v11.5.1+ for trusted publishing support
   - Skip logic prevents re-publishing immutable versions
   - Dry-run option for testing the workflow without actual publishing
   - `defaults.run.working-directory: mission-control` sets cwd for all steps
   - SDK loop uses subshell `( cd ... )` to avoid cwd drift between iterations
