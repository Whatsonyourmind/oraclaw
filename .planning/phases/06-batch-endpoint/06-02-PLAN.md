3. Add a batch-specific metering onResponse hook AFTER the existing Stripe meter hook. This hook:
   - Only fires when `request.isBatchRequest === true` AND `request.billingPath === 'stripe'` AND `request.stripeCustomerId` exists AND `reply.statusCode < 400`
   - Emits ONE meter event to a batch-specific meter event name (`process.env.STRIPE_BATCH_METER_EVENT_NAME || 'api_calls_batch'`)
   - Uses `value: String(request.batchSize)` -- this is a WHOLE number (count of calls in batch). The Stripe meter is configured with a unit price at 50% of the per-call rate to achieve the discount.
   - Follows the same fire-and-forget pattern as the existing meter hook: `.catch((err) => request.log.error(...))`
   - Uses identifier: `${request.id}-batch-${Date.now()}`

The hook should be inline in index.ts (same pattern as the existing meter hook registration on line 144-148), NOT a separate file. It reuses the same `stripe` import already in index.ts.

```typescript
// Batch metering: emit usage at 50% rate for batch requests
// Separate meter event name configured in Stripe with 50% unit pricing
server.addHook('onResponse', async (request, reply) => {
  if (
    request.url.startsWith('/api/v1/') &&
    request.isBatchRequest &&
    request.billingPath === 'stripe' &&
    request.stripeCustomerId &&
    reply.statusCode < 400
  ) {
    const batchEventName = process.env.STRIPE_BATCH_METER_EVENT_NAME || 'api_calls_batch';
    stripe.billing.meterEvents
      .create({
        event_name: batchEventName,
        payload: {
          stripe_customer_id: request.stripeCustomerId,
          value: String(request.batchSize || 0),
        },
        identifier: `${request.id}-batch-${Date.now()}`,
      })
      .catch((err) => {
        request.log.error({ err }, 'Stripe batch meter event failed');
      });
  }
});
```

CRITICAL: The batch meter hook uses WHOLE number values only (Stripe constraint from research). The 50% discount is achieved by configuring the Stripe meter's unit price at half the per-call rate, NOT by sending fractional values.
CRITICAL: The existing meter hook skip (Part A) must be in place BEFORE registering the batch route, or batch requests will be double-metered.
CRITICAL: Do NOT create a separate meter-usage-batch.ts file. Keep the hook inline in index.ts consistent with how the existing meter hook is registered.
