1. In `src/middleware/auth.ts`, add two optional fields to the existing `FastifyRequest` interface augmentation:
   - `isBatchRequest?: boolean` -- flag for the existing meter-usage hook to skip batch requests
   - `batchSize?: number` -- number of calls in the batch, used by Plan 02 for batch metering

2. Create `src/routes/oracle/api-batch.ts` as a Fastify plugin (same `export default async function` pattern as api-public.ts):

   a. **Types:**
   ```typescript
   interface BatchCall {
     algorithm: string;
     params: unknown;
   }
   interface BatchRequestBody {
     calls: BatchCall[];
   }
   interface BatchCallResult {
     algorithm: string;
     status: 'success' | 'error';
     data?: unknown;
     error?: { type: string; title: string; status: number; detail: string };
   }
   interface BatchResponse {
     results: BatchCallResult[];
     meta: { total: number; succeeded: number; failed: number };
   }
   ```

   b. **ALGORITHM_DISPATCH** lookup table: `Record<string, (params: unknown) => Promise<unknown>>` mapping all 17 algorithm endpoint path names to async handler functions. For each algorithm, replicate the body-parsing and execution logic from api-public.ts inline route handlers. The key is the path segment after `/api/v1/` (e.g., `'optimize/bandit'`, `'score/convergence'`). Export this for testability.

   c. **Route handler** for `POST /api/v1/batch`:
   - Validate `body.calls` is a non-empty array with length <= 20. Use `sendProblem` with `ProblemTypes.VALIDATION` for errors.
   - Execute all calls via `Promise.allSettled(body.calls.map(...))`. For each call:
     - Look up algorithm in `ALGORITHM_DISPATCH`. If unknown, reject with descriptive error.
     - Call the handler function with `call.params`.
   - Map settled results to `BatchCallResult[]` preserving order. Fulfilled = `{ status: 'success', data }`. Rejected = `{ status: 'error', error: { type, title, status, detail } }` using RFC 9457 format inline (not sendProblem, which writes to reply).
   - Set `request.isBatchRequest = true` and `request.batchSize = body.calls.length` for downstream hooks.
   - Return `{ results, meta: { total, succeeded, failed } }` with HTTP 200 always (even if all fail).

   d. Keep api-public.ts UNTOUCHED. The dispatch map imports the same service-level functions and replicates handler logic.

   CRITICAL: Use `Promise.allSettled` (NOT `Promise.all`) for partial-failure handling.
   CRITICAL: Return HTTP 200 even when all calls fail -- the batch endpoint itself succeeded.
   CRITICAL: The dispatch map should NOT use `app.inject()` (avoids hook re-execution per research pitfall 3).
