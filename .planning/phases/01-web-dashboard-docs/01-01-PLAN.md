# Plan 01-01: Next.js Dashboard Scaffold & Algorithm Catalog

## Context

OraClaw needs a public web presence for developer onboarding. Currently the API has Scalar docs at /docs but no landing page, algorithm catalog, or getting-started content. This plan creates the Next.js app scaffold with the algorithm catalog page showing all 19 algorithms with descriptions, categories, pricing, and input/output schemas.

## Requirements Addressed

- WEB-01: Public Next.js site with algorithm catalog
- DX-01: Scalar interactive playground (carried from v21.0)

## Tasks

### Task 1: Next.js App Scaffold

Create a new Next.js 15 app in `mission-control/apps/web/` with:
- App Router (app/ directory)
- TypeScript strict mode
- Tailwind CSS for styling (dark theme matching OODA colors)
- Layout with header (OraClaw logo, nav: Algorithms, Docs, Pricing, Try It) and footer
- Homepage with hero section: "Decision Intelligence as an API" tagline
- Meta tags and favicon

### Task 2: Algorithm Catalog Page

Create `/algorithms` page displaying all 19 algorithms organized by category:
- **Optimize**: Multi-Armed Bandit, Contextual Bandit (LinUCB), Genetic Algorithm, CMA-ES, Simulated Annealing
- **Simulate**: Monte Carlo Simulation, Scenario Planning
- **Solve**: Constraint Optimizer (LP/MIP), Schedule Optimizer, A* Pathfinding
- **Analyze**: Decision Graph, Ensemble Model, Portfolio Risk (VaR/CVaR), Correlation Matrix
- **Predict**: Bayesian Inference, Markov Chain, Q-Learning, Time Series Forecast
- **Detect**: Anomaly Detection, Convergence Scoring, Attention Mechanism

Each algorithm card shows: name, category tag, one-line description, pricing (per-call USDC), and a "Try It" link.

### Task 3: Scalar API Playground Embed

Create `/docs` page embedding Scalar API Reference pointed at the live API's OpenAPI spec:
- Fetch spec from `https://oraclaw-api.onrender.com/docs/json` (or local dev fallback)
- Full Scalar interactive playground with authentication support
- Pre-filled example requests for each endpoint
