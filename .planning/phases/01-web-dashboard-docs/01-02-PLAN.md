# Plan 01-02: Interactive Try-It Forms & Getting Started Guide

## Context

The algorithm catalog from Plan 01-01 shows what's available. This plan adds interactive try-it forms so developers can call algorithms directly from the browser, plus a getting-started guide that walks through API key creation to first result in under 2 minutes.

## Requirements Addressed

- WEB-02: Interactive try-it forms with pre-filled examples and live API results
- WEB-03: Getting-started guide completable in under 2 minutes

## Tasks

### Task 1: Interactive Try-It Forms

Create `/try/[algorithm]` dynamic route with:
- Pre-filled JSON editor (textarea) with example input for each algorithm
- "Run" button that POSTs to the live API (configurable base URL)
- Response display with formatted JSON output, timing, and status
- API key input field (optional -- free tier works without one)
- Algorithm-specific examples that demonstrate the key value proposition

Cover the most impactful algorithms first:
1. `/try/bandit` — Multi-Armed Bandit with 3 arms
2. `/try/montecarlo` — Monte Carlo with normal distribution
3. `/try/bayesian` — Bayesian belief update
4. `/try/evolve` — Genetic algorithm optimization
5. `/try/anomaly` — Anomaly detection on sample data
6. `/try/pathfind` — A* pathfinding on a small graph

### Task 2: Getting Started Guide

Create `/getting-started` page with:
1. **Step 1**: Get your API key (link to Unkey dashboard or instructions)
2. **Step 2**: Make your first call (curl + JavaScript + Python examples)
3. **Step 3**: See the result (annotated response explanation)
4. Code snippets use `@oraclaw/bandit` SDK as the "hello world" example
5. Pricing summary: free tier = 100 calls/day, no key needed

### Task 3: Navigation & Polish

- Wire all pages into the nav header
- Add breadcrumbs to subpages
- Footer with GitHub link, npm link, API status badge
- Mobile-responsive layout verification
