### Stories 1.1-1.6: Database Schema (COMPLETE)
Implemented all ORACLE database tables in schema.sql:

**OBSERVE Module (1.1):**
- oracle_data_sources: Data sources for signal detection (calendar, email, tasks, etc.)
- oracle_signals: Signals with urgency, impact, confidence, status tracking
- oracle_signal_clusters: Groups of related signals with combined metrics
- oracle_signal_cluster_members: Junction table for cluster membership
- oracle_anomaly_patterns: Detection rules with baseline data and thresholds

**ORIENT Module (1.2):**
- oracle_contexts: Strategic contexts with situation summaries, key factors, recommendations
- oracle_horizons: Multi-horizon planning (immediate/today/week/month) with goals and actions
- oracle_correlations: Entity relationships with strength DECIMAL(3,2), direction, evidence
- oracle_assessments: Risk/opportunity assessments with impact levels and mitigations

**DECIDE Module (1.3):**
- oracle_decisions: Decisions with status (pending/analyzing/decided/executed/cancelled)
- oracle_decision_options: Options with pros, cons, estimated outcomes, scoring
- oracle_simulations: Monte Carlo results with iterations, mean, stdDev, percentiles in JSONB
- oracle_stakeholder_inputs: Stakeholder tracking with approval/opinion/constraint types
- oracle_critical_paths: Execution paths with dependencies, bottlenecks, parallel tracks

**ACT Module (1.4):**
- oracle_execution_plans: Plans with health score, progress (DECIMAL 5,2), step counts
- oracle_execution_steps: Steps with status (pending/in_progress/completed/blocked/skipped)
- oracle_progress_updates: Update tracking with deltas and status changes
- oracle_plan_adjustments: Dynamic replanning with change tracking
- oracle_execution_outcomes: Final results with variance analysis
- oracle_learnings: Captured lessons linked to outcomes for feedback loop

**Probability Engine (1.5):**
- oracle_predictions: Predictions with confidence (DECIMAL 3,2), factors, decay rate
- oracle_outcomes: Actual outcomes for Bayesian updates
- oracle_calibration: Brier score tracking, accuracy by bucket, alpha/beta for Beta distribution
- oracle_user_patterns: Behavioral patterns in JSONB (time_of_day, completion_rate, etc.)

**Environment Awareness (1.6):**
- oracle_environment_snapshots: Device state, location, battery, network, calendar context
- oracle_context_nodes: Graph nodes (person, project, task, event, location, etc.)
- oracle_context_edges: Graph edges with strength, direction, reinforcement tracking
- oracle_ghost_actions: Pre-prepared actions with approval workflow status

**Key Patterns Used:**
- UUIDs for all primary keys: `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
- Timestamps: `created_at TIMESTAMPTZ DEFAULT NOW()`, `updated_at TIMESTAMPTZ DEFAULT NOW()`
- JSONB for flexible data: `metadata JSONB DEFAULT '{}'`
- Confidence as DECIMAL(3,2) with CHECK constraints
- RLS enabled on all tables with user-based policies
- Indexes on user_id, created_at, status, type, and foreign keys
- Updated_at triggers using existing update_updated_at_column() function

### Stories 5.1-5.3: Oracle Services (COMPLETE)
Created all ORACLE backend services in apps/api/src/services/oracle/:

**Monte Carlo Simulation (5.1) - monteCarlo.ts:**
- MonteCarloSimulator class with multiple distribution support
- Distributions: normal, lognormal, uniform, triangular, beta, exponential
- run() method with configurable iterations (default 1000)
- Timeout protection (10s) and iteration cap (2000)
- Returns: mean, std_deviation, percentiles (p5/p25/p50/p75/p95), distribution

**Probability Engine (5.2) - probability.ts:**
- ProbabilityEngine class for Bayesian updates
- updateBayesian(): Beta(α, β) → Beta(α + success, β + failures)
- applyTimeDecay(): P(t) = P₀ * e^(-λt) for confidence decay
- calculateBrierScore(): (1/N) * Σ(forecast - outcome)²
- updateCalibration(): Maintains accuracy buckets for calibration
- combineForecasts(): Weighted average of multiple predictions

**Cache Service (5.3) - cache.ts:**
- OracleCacheService with LRU eviction
- TTL by category: predictions (5min), simulations (30min), calibration (1hr), gemini (15min)
- getOrSet() and getOrSetByCategory() for cache-aside pattern
- Cache stats and expired entry cleanup
- hashObject() utility for cache key generation

- Updates pattern boost records
